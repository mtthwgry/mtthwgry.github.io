In this blog post, I will take a high-level look at the differences between Ruby and JavaScript by providing some examples for comparison. The truth of the matter is that these two languages have a lot in common, but have syntax and structure that vary greatly. That said, if you know Ruby pretty well then JavaScript shouldn't challenge you too much.

Ruby Methods vs. JavaScript Function

Ruby calls them methods, while JavaScript calls them functions. Methods and functions are synonymous with each other. See below examples of a method definition in Ruby, and a function definition in JavaScript. #Ruby method definition: def add(num1, num2) num1 + num2 end //JavaScript function definition: var add = function(num1, num2) { return num1 + num2; }; The main difference here is the way a method/function is defined. In Ruby, the keyword def is used, but in JavaScript you need to set a variable using the var keyword and then set that equal to the function you build. Another notable difference are the curly braces, {}, which enclose the body of a function in JavaScript. Ruby uses the keywords def followed by the method name (and arguments, if any) the body of the method begins on the next new line, and is closed by the keyword end. Lastly, Ruby has an implicit return for methods, meaning that the last line evaluated in the method will be the returned value. JavaScript doesn't have an implicit return, so you must use the return keyword.

Ruby Hashes vs. JavaScript Object

Is a Ruby hash the same as a JavaScript Object? The answer is no, but they are very similar. An object in JavaScript is a collection of attributes and functions that are unique to that given instance. Yes, you read correctly, an object in JavaScript can contain attributes and functions. A Ruby hash is just a set of key-value pairs, but isn't the syntax the same as a JavaScript object? See the examples below: #Ruby hash: car = { color: "black", make: "Tesla", model: "Model S" } //Javascript object: var car = { color: "black", make: "Ford", model: "Model T" }; You can see that the instantiations of these two objects are identical (except for the var and ; in JavaScript). This can be confusing, because a JavaScript object looks like a Ruby hash, but it actually behaves more like a Ruby object. See below: //JavaScript file console.log(car.color); console.log(car.make); console.log(car.model); //Console Output //black //Ford //Model T That's crazy! You can access the attributes of a JavaScript object using dot notation...just like with Ruby objects. So how do you store a function in an object? Simple. car.drive = function(distance) { console.log(this.model + " just drove " + distance + " miles."); }; The other difference is that JavaScript objects don't have the built-in methods that Ruby hashes do.
